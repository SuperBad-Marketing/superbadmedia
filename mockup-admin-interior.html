Admin Interior · reference sheet · 2026-04-15

The room we actually run the business from.

Every admin interior built before today ships generic — flat black, thin outlined cards, all DM Sans, no warm depth. This sheet is the fix: a shared anchor for every surface behind the nav.

What this is. A compositional reference — not a design system page (that's /lite/design). It shows how the existing tokens assemble into the primitives admin interiors keep reaching for: stage chips, data cards, table rows, section headers, toolbars, empty states, the Won moment, the stale affordance. If you're building an admin interior from admin-polish-1 onward, this file is binding §2a reading alongside the spec.

What this isn't. Per-route mockups. Those follow — sliced once this anchor is in hand. If a future admin surface needs a primitive that isn't here, cut a surface-specific mockup first and extend this sheet.

§ 01 · palette recap

Warm neutrals, brand accents, one success green.

§ 02 · typography recap

Four voices, each with a job.

Admin interiors lean on the same four typefaces as the rest of Lite. No DM-Sans-only screens. If a surface has no Righteous eyebrow and no Black Han Sans display, it reads as a Linear-clone — that's the failure mode this sheet exists to prevent.

Every admin page opens with a voice, not a breadcrumb.

§ 04 · toolbars

Search on the left, actions on the right, filters in between.

Toolbars sit on surface-2, not floating in the void. Primary action is brand-red and Righteous-capped. Secondary actions are ghost outlines. Filter chips animate with the house spring on activate.

§ 05 · stage + status chips

Chips earn colour; everything else is neutral.

Pipeline stages get a cool-to-hot accent walk (pink → orange → warn → red → success). Deal outcomes are their own palette. Stale is always a dashed outline over transparent — it never takes colour because stale is the absence of motion.

Tables whisper; they don't grid.

§ 08 · empty states

Empty is a voice moment, not a shrug.

Every empty state has an eyebrow, a Black Han Sans line, one sentence of body, and a mutter. Never "No items found." That's the Linear default — and the single clearest signal a surface wasn't built against this sheet.

§ 10 · stale affordances

Stale is dashed, dimmed, and breathing slowly.

When a deal, thread, or integration hasn't moved inside its stage threshold, it picks up a dashed border, muted titling, and a 3.2s halo pulse. The pulse is slow enough to read as ambient worry, not an alarm. feedback_motion_is_universal applies — but gently.

§ 11 · alerts + banners

Three temperatures. No yellow-triangle clipart.

Warm gradient for attention-needed (new warm upsell, expiring quote). Muted green for good news (paid, integration healthy). Cool dark for informational (scheduled job, kill-switch state). Who / What / Why tri-line structure — why is always italic.

§ 13 · binding rules

What every admin-polish session must hold.

These aren't suggestions. admin-polish-1..N G10 parity checks run against these rules and against the primitives above.

  1. Page headers use Black Han Sans, always — with a voiced deck.
    A DM-Sans "Pipeline" h1 with "Manage your deals" subtitle is a failure signal. Voice lives in the deck; visual weight lives in the title.
  2. Eyebrows, chips, and metric headers are Righteous, all-caps, tracked ≥1.5px.
    If a section or category label is DM Sans, it's reading as body copy — not as structure.
  3. Cards sit on surface-2 with --inner-highlight. No hairline outlines over flat black.
    The inner highlight is what gives admin interiors warmth. Thin borders over #18181A read as shadcn/Linear — the exact thing this sheet replaces.
  4. Stale is dashed + muted + slow-pulsing. Never coloured.
    Stale means absence of motion. Giving it orange or red makes it read as urgent; it's the opposite — it's dormant. The dashed outline is the tell.
  5. Won / paid moments earn the BHS treatment. Nothing else does.
    If every successful save triggers a gradient callout, BHS stops meaning anything. Reserve for genuine closures — deal won, invoice paid, wizard completed, onboarding finished.
  6. Empty states are voiced. "No items" is a bug.
    Every empty state gets an eyebrow + BHS title + body + mutter. If you can copy-paste an empty state between surfaces without editing, it's too generic.
  7. Mutter lines live in Playfair italic, brand-pink. One per surface, max.
    The mutter is the quiet SuperBad joke — overuse drowns it. One per page header, plus optional inline passive-line footers on multi-column layouts.
  8. Primary CTA is brand-red, Righteous-capped, with --inner-highlight + glow. Secondary is ghost.
    One brand-red action per surface. If two primaries compete, one of them is actually secondary — make it ghost and stop the visual tie.
  9. Hover states animate with --spring over 160–200ms. No CSS default easing.
    feedback_motion_is_universal. Every hover, every stage change, every filter activate — houseSpring. Default ease-in-out is the generic tell.
  10. Numeric columns and counts use Righteous with letter-spacing — never plain DM Sans tabular.
    Numbers in Righteous read as earned metrics; numbers in DM Sans read as form input. Admin interiors are for reading, not entry.
