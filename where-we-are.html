Suggested Upgrades

A curated shortlist — twelve additions that slot cleanly into the planned architecture without breaking anything, and would push Lite toward best-in-class. Tick the ones you want in.
Universal Audit Log
Operational Safety
Every admin action — card dragged, deal edited, email sent, client added — gets a timestamped, immutable row. Compounds into forever-value. You never have to ask "who did what, when, why" again. Quiet insurance for the day a client disputes something.
The Sales Pipeline spec already defines an append-only activity log with nullable FKs. Generalise that same table to every mutation at the Drizzle layer. Zero risk, one pattern.
Global Undo Toast
Premium Polish
Every destructive action gets a 10-second "undo" toast. Deleted a card by accident? Tick. Moved a deal to Lost? Tick. It's the single cheapest thing you can build that makes software feel trustworthy — Notion, Linear, and Superhuman all win on this detail alone.
Soft-delete pattern at the data layer, a shared toast component, one undo adapter. Pairs naturally with the audit log above.
Safe-To-Send Gate
Operational Safety
Every outbound email — outreach, wizard confirmations, newsletter, client automations — flows through a single canSendTo(recipient) check. Suppression list, bounce history, unsubscribe flags, frequency caps. One primitive. Protects deliverability forever. Impossible to retrofit if you get it wrong early.
A single middleware wrapped around the Resend adapter. Every send path imports it, nothing ships without it. Non-negotiable discipline, not a feature you toggle.
Timezone-Correct Everything
Operational Safety
Store every timestamp in UTC, display every timestamp in the viewer's local timezone. Add a timezone field to user records. Cheap to enforce day one. Catastrophic to retrofit once a client in Perth says "why does my invoice say Tuesday when I paid Wednesday."
A date-display utility used everywhere, one column added in the user schema. A Phase 4 build-plan rule, not a feature.
Morning Brief · Listen In The Car
Founder Protection
The daily cockpit's narrative morning brief already exists in spec. Pipe it to a 6:30am email with a ~30-second audio file attached — voice-generated, in SuperBad tone. You hear the state of the business before you hit the studio. Nothing else does this well.
Uses Claude (already in stack) for the text, a TTS API for the voice, Resend (already in stack) for delivery, a daily cron job. Zero new architectural surface.
Quiet Mode Outreach Window
Operational Safety
Automated outreach (cold email, follow-ups, auto-nudges) can only send between 8am–6pm Melbourne local, never on Aus public holidays. Protects sender reputation, respects recipients, and removes the "oh no I sent that at 11pm" anxiety. Set once, enforced forever.
A single gate in the channel adapter layer. Holiday calendar is a static JSON file updated yearly. Rejected sends queue for the next safe window.
Monday Founder-State PDF
Founder Protection
Every Monday 7am, a branded one-page PDF lands in your inbox. MRR, ARR, deals by stage, clients that went quiet, overdue invoices, content engine output, weekly wins. Print it, drink coffee, decide the week.
Puppeteer HTML-to-PDF is already earmarked in the stack. Template is one React component. Cron job, email, done. Composes existing data — no new models.
Brand-Voice Drift Check
Intelligence Layer
Every LLM-generated artifact — outreach email, blog draft, newsletter, reply suggestion — passes a final "does this sound like the Brand DNA profile?" check before you ever see it. Claude grades the draft against the signal tags and flags drift. The difference between a platform that generates content and a platform that generates your content.
One extra Claude call per generation, reading the client's Brand DNA profile as context. Natural extension of the "Brand DNA as perpetual context" pattern already locked in memory.
Sentiment-Aware Reply Drafts
Intelligence Layer
Prospect replies. Claude reads the tone — warm / cold / curious / objection / excited — tags it, and drafts a reply in matching register. Appears inline in the unified inbox as a suggested reply, not auto-send. Collapses the "what do I say back" friction that kills momentum on live leads.
Slots into the Unified Inbox spec as a single draft-suggestion module. Uses the same Claude primitive as outreach generation. No new infra.
Global Semantic Search
Intelligence Layer
Type anything into the top bar — "that clinic in Brighton", "the guy who wanted three shoot days", "the blog post about rebranding" — and get matches across contacts, deals, emails, blog drafts, and notes. Not keyword search. Meaning search.
Embeddings stored alongside each record on write (one vector column per entity table). At query time, embed the input and nearest-neighbour search. SQLite's sqlite-vec extension handles it — no external vector DB needed.
Client Portal "Trailer" On First Visit
Premium Polish
The first time a client opens their portal link, they see a ten-second branded animated intro — their name, their brand colours, a dry welcome in SuperBad voice, then the portal fades in. One-time only. The kind of moment people screenshot and send to friends. Directly serves the "customer-facing surfaces feel individual, not communal" memory.
A first_portal_visit_at timestamp on the client record + a Framer Motion intro component (already in stack). Runs once, then never again.
One-Click Client Data Export
Premium Polish
One button on any client record: "Export everything Lite knows about them." Outputs a branded ZIP — CSV of their data, PDFs of their invoices and quotes, a list of linked external folders. A trust signal if they ever ask, a migration safety net, and quietly premium as a receipt of how well you handle their business.
Composes existing PDF generation and data queries into a single handler. No new data, just a shape-change. Zero architectural risk.
05 · Honest Reality Check

The Parts That Could Bite.

Not to be scary — just so you know where to watch the road.

  • The Brand DNA question bank is creative work, not code. Fifty to a hundred indirect-signal questions per bank, with branching logic and a signal-tag taxonomy. If the questions are generic, the profile is generic, and every downstream LLM call inherits that mediocrity. This is the single highest-stakes creative session in Phase 3. Worth loading the brand-voice skill the morning of and taking it slow.
  • Content engine output in your voice is the whole ballgame. If Claude's blog drafts sound like LinkedIn boilerplate, the rejection chat becomes a chore and the engine dies on the vine. The one-gate review only works if the base output is already 80% there. Spec-session priority is prompt engineering, not feature surface.
  • Cold email deliverability is infrastructure, not code. Subdomain setup, SPF / DKIM / DMARC, warming the sender, staying under volume thresholds. Get it wrong and outreach silently goes to spam for months without you knowing. Worth a full setup wizard of its own and Safe-To-Send Gate above.
  • DNS access for superbadmedia.com.au is still unresolved. Everything URL-related blocks on this — client portals, cold-email subdomain, the eventual /lite cutover, the Content Engine's /blog routing. Resolve it early in Phase 5, not late.
  • Thirteen specs plus a build phase is a lot of sessions. Handoff-note hygiene between sessions is the entire difference between shipping and re-discovering. One slipped handoff costs a full rediscovery session. Tick a spec here only after the handoff note is written.
