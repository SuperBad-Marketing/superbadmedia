# Parked — Marketing Site Brainstorm

_Items surfaced during the Phase 1 visual/experiential brainstorm that belong to later phases. Each should be resolved before the relevant build session opens._

---

## P1 · Self-shot portfolio list (anchors + supporting)

**What:** Generate a numbered shoot list Andy can execute himself — no client permission needed — that gives the Work door launchable content.

**Must include:**
- **3–4 anchor concepts** conceived as full anchor projects: video + stills + written brief + pull quote + outcome paragraph. These are the flagship pieces on the Work grid.
- **8–12 supporting concepts** at lighter production cost: video reel + minimal credits only.
- Each entry: concept, Melbourne-shootable location type, shoot complexity (solo achievable), subject type (food / retail / trades / hospitality / product / lifestyle), why-it-sells-the-agency.

**Constraints:**
- No reliance on real client footage/photos.
- Video is primary in every entry; stills where the concept warrants them.
- Visual direction must pass the Warm Filmic (A) grade cleanly — avoid scenes that depend on colour treatments the locked grade won't support.

**When:** After visual brief lock, before first Work-door build session.

---

## P2 · Performance marketing proof-of-work strategy

**What:** How does SuperBad demonstrate performance marketing / media buying credibility on launch, without access to real client campaign data or permission?

**Candidate approaches to resolve between:**
- Synthetic case study on SuperBad's own growth (the agency as its own case study).
- A public, live campaign running on SuperBad as the advertiser, with results updated in near-real-time.
- Anonymised past-work results — carefully scrubbed.
- A "how we think about performance" essay in the Thinking section doing the conceptual heavy lifting while real case studies accumulate.
- Some combination.

**Constraints:**
- Performance work does not appear in the Work portfolio (locked in Q3).
- Lives in Services page (inside the Trial funnel), in the 6-week plan, and in Thinking.
- Can't look desperate or padded — a thin performance section is worse than no performance section.

**When:** Before Services page spec is drafted (inside the Start-a-Trial funnel sub-brainstorm).

---

## P3 · Start-a-Trial funnel — full sub-brainstorm

**What:** The Start-a-Trial door contains Services content, pricing, trust/social proof, and the booking flow. That's a mini-site. Deserves its own brainstorm session rather than being designed in passing during the sitemap pass.

**Scope of the sub-brainstorm:**
- Entry: what the door looks like from the homepage click.
- Services presentation: how creative + performance service lines are described, without making performance look thin.
- Pricing: whether tiers are shown publicly, at what depth, with what framing.
- Trust content: case studies? testimonials? Andy's own voice? How much About content is folded in?
- Booking flow: how the trial shoot booking actually happens (form? calendar? something else?).
- 6-week plan preview: is it mentioned here? How much is shown?
- Pre-filled experience: memory says outreach research pre-fills trial shoots — how does this render for a visitor who arrived via outreach vs. cold?
- Hand-held setup: memory says setup is never self-served — how does that apply to the trial entry point?

**Dependencies:**
- P2 must resolve first (so we know how to show performance).
- Brand DNA questionnaire logic (already in Lite backlog) needs to be aligned — trial shoot unlocks the questionnaire per memory.

**When:** After sitemap lock, before build plan finalises.

## P4 · Remaining micro-experiential calls

Defer to per-section specs, not brainstorm questions in isolation:

- **Cursor** — default or custom; if custom, what register.
- **Scroll progress affordances** — chapter markers inside essays, progress indicator on anchor project pages.
- **Footer treatment** — what lives in it; how it closes the site on each page.
- **Delight moments / easter eggs** — memory locks the philosophy (80% ambient, 20% MGS-style); specific placements decided per surface.
- **Empty-state illustrations** — whether utility states get any visual treatment beyond typography.

**When:** Inside each section spec during Phase 3.
